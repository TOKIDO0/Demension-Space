# 网站部署指南（小白版）

## 一、网站部署的基本概念

### 什么是部署？
部署就是把你的网站代码放到互联网上，让其他人可以通过网址访问。

### 前台和后台的区别
- **前台页面**：给客户看的，展示作品、联系方式等（任何人都可以访问）
- **后台管理页面**：给你自己用的，管理作品、查看留言等（需要登录才能访问）

## 二、部署方案推荐

### 方案1：使用 Vercel（推荐，最简单）

**优点：**
- 免费
- 操作简单
- 自动部署
- 速度快

**步骤：**

1. **注册账号**
   - 访问 https://vercel.com
   - 用 GitHub 账号登录（如果没有，先注册 GitHub）

2. **上传代码**
   - 在 GitHub 创建一个新仓库（repository）
   - 把你的代码上传到 GitHub
   - 在 Vercel 中导入这个仓库

3. **配置环境变量**
   - 在 Vercel 项目设置中添加：
     - `SUPABASE_URL`
     - `SUPABASE_ANON_KEY`
     - `NTFY_TOPIC`（如果需要）

4. **部署**
   - Vercel 会自动部署
   - 你会得到一个网址，比如：`your-site.vercel.app`

### 方案2：使用 Netlify（类似 Vercel）

步骤和 Vercel 类似，访问 https://netlify.com

## 三、后台管理页面的安全

### 问题：部署后别人能访问后台吗？

**答案：可以访问页面，但无法登录。**

### 为什么？
- 后台页面需要登录才能使用
- 只有你知道账号密码
- 即使别人打开后台页面，没有账号也进不去

### 额外安全措施（可选）

1. **隐藏后台地址**
   - 不要在前台页面链接到后台
   - 使用复杂的后台地址，比如：`your-site.com/admin-secret-2024`

2. **IP 白名单**（高级）
   - 只允许特定 IP 访问后台
   - 需要服务器支持

3. **双重验证**
   - 在 Supabase 中启用邮箱验证
   - 添加手机验证码

## 四、部署后的操作

### 前台页面
- 任何人都可以通过网址访问
- 可以查看作品、提交留言、查看评价

### 后台管理
- 访问：`your-site.com/admin/dashboard.html`
- 需要登录你的管理员账号
- 只有你能管理内容

## 五、常见问题

**Q: 部署后需要修改代码吗？**
A: 不需要，代码会自动从 GitHub 部署。

**Q: 数据库需要单独部署吗？**
A: 不需要，Supabase 已经在云端了。

**Q: 部署是免费的吗？**
A: Vercel 和 Netlify 都有免费套餐，够用。

**Q: 可以绑定自己的域名吗？**
A: 可以，在 Vercel/Netlify 设置中添加你的域名。

## 六、快速开始（3步）

1. **注册 Vercel**：https://vercel.com
2. **上传代码到 GitHub**
3. **在 Vercel 导入并部署**

完成！你的网站就可以被访问了。
